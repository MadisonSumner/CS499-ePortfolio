Enhancement One: Software Design & Engineering

Course: CS-499 Computer Science Capstone

Original Course: CS 360

Original Development Period: September–October 2025

Artifact Description

The selected artifact is an Android inventory management application originally developed for CS 360. The application includes a login interface, an inventory display page showing item names, quantities, and creation dates, as well as a notification mechanism designed to alert users when stock levels are low.

While the initial implementation was functional, it lacked a clear separation of concerns and domain-focused architecture. This limitation provided an opportunity to enhance the system through structural and architectural improvements.

Justification for Inclusion

This artifact was chosen for inclusion in my ePortfolio because it reflects a practical software solution while offering meaningful opportunities to demonstrate software design and engineering competencies. The application integrates database interactions, user interface components, and business logic, making it well suited for architectural refinement.

The primary enhancement involved refactoring the application to follow domain-driven design (DDD) principles. This refactor introduced clearly defined repositories and contexts to separate data persistence from domain logic. Domain objects were established with encapsulated validation using DomainResult, improving structure and reliability.

Additionally, the user interface was reorganized to align more closely with domain workflows, particularly within the inventory editing functionality. This restructuring illustrates effective collaboration between the domain, repository, and UI layers. The resulting system emphasizes clean architecture, maintainability, extensibility, and improved testability.

Course Outcome Alignment

This enhancement directly supports the program outcome related to software design and engineering. The refactoring demonstrates my ability to design, implement, and refine a software system using established architectural practices while maintaining separation between domain, infrastructure, and presentation layers.

Reflection on the Enhancement Process

Throughout the enhancement process, I gained a deeper appreciation for planning around single-responsibility principles and well-defined interface contracts prior to implementation. Refactoring the original codebase, which lacked sufficient documentation and structure, required deliberate analysis and systematic reorganization.

Research into domain-driven design principles guided the implementation of repositories, contexts, and domain entities in a way that integrated seamlessly with the existing system. A key challenge involved maintaining backward compatibility to preserve original functionality while improving architectural clarity.

Overall, this enhancement strengthened my understanding of system architecture, domain modeling, and the practical application of structured design principles in a real-world software environment.
