Enhancement Two: Algorithms & Data Structures

Course: CS-499 Computer Science Capstone

Original Course: CS 360

Original Development Period: September–October 2025

Artifact Description

The selected artifact is an Android inventory management application originally developed for CS 360. It includes a login page, an inventory display page showing item names, quantities, and creation dates, and a notification system intended to alert users when stock levels are low.

While functional, the original application lacked dynamic automation to help users manage low-stock items effectively.

Justification for Inclusion

This artifact was chosen because it provides a real-world context to demonstrate algorithmic and data structure skills while enhancing an existing application. The enhancement introduces a ShoppingListService that automatically identifies items below user-defined thresholds and dynamically maintains a prioritized shopping list.

This feature uses a PriorityQueue data structure to efficiently sort items based on how critically low their stock is relative to their thresholds. The service evaluates all inventory items whenever updates occur, adding items below threshold and removing restocked items.

Encapsulating this logic in a dedicated service separates algorithmic decision-making from the UI and database layers, emphasizing clean architecture and demonstrating practical application of advanced data structures.

Course Outcome Alignment

This enhancement aligns with course outcomes by demonstrating the ability to design and evaluate computing solutions using algorithmic principles and appropriate data structures. Implementing the shopping list service illustrates competency in algorithmic design, specifically in using a priority queue to efficiently manage and sort items based on stock levels relative to thresholds.

The work reflects progress toward employing well-founded computing techniques that balance functional value with efficiency and maintainability.

Reflection on the Enhancement Process

Implementing the shopping list feature gave practical experience with priority queues in Java and applying them in a real-world scenario. I learned to dynamically evaluate inventory items and maintain a list reflecting low-stock priorities, reinforcing my understanding of algorithmic logic and collection management.

A major challenge was updating the database to include a new lowStockThreshold column. Initially, the onCreate() method was not invoked on fresh installs, requiring careful management of database versions and schema upgrades to ensure reliable functionality.

Overall, this enhancement strengthened my ability to integrate data structures into functional applications, design services encapsulating algorithmic logic, and anticipate practical challenges in production-like environments.
