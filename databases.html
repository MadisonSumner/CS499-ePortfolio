Enhancement Three: Databases

Course: CS-499 Computer Science Capstone

Original Course: CS 360

Original Development Period: September–October 2025

Artifact Description

The selected artifact is an Android inventory management application originally developed for CS 360. It includes a login page, an inventory page that displays item names, quantities, and creation dates, and a notification system to alert users when stock levels are low.

While functional, the original application lacked structured organization for categories and efficient relational queries.

Justification for Inclusion

This artifact demonstrates my ability to implement relational database enhancements in a production-like application. Prior architectural improvements ensured that domain logic was agnostic to the database, allowing database enhancements without affecting existing functionality.

All table definitions are organized under their respective contracts — for example, categories are defined by CategoryTableContract and inventory items by InventoryItemTableContract. These contracts are applied in InventoryDbContext, which manages all database setup.

For this milestone, I added a Category table linked to the Inventory table via a foreign key and created an index on the foreign key to improve query performance when filtering by category. The Category table is mapped to a domain model, and inventory items can now be added, edited, and filtered by category. These enhancements demonstrate practical database skills, including table creation, foreign key relationships, indexing, and integration of database structures with application features.

Course Outcome Alignment

This enhancement demonstrates competency in applying well-founded database techniques to implement functional solutions. It aligns with outcomes for building structured data environments that support decision-making, delivering technically sound solutions, and using innovative database practices.

Consolidating planned outcomes into this single artifact demonstrates database design, management, and integration within a real-world application.

Reflection on the Enhancement Process

Enhancing the database involved applying migrations for the Category_ID column to the existing Inventory table. A significant challenge was maintaining data integrity while updating the schema on existing installations, which was solved by updating the onUpgrade method in InventoryDbContext.

Another challenge was that SQLite did not support foreign keys by default. After research, I enabled this feature with an onOpen function. These solutions strengthened my understanding of relational database theory, foreign keys, indexing, and maintaining data integrity in a functional application.

Overall, this enhancement reinforced the importance of aligning database design with application functionality and demonstrated practical skills in implementing robust, maintainable, and scalable database solutions.
